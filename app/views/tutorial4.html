Tutorial four - Show and hiding things based on answers

Showing and hiding things based on the inputs of users answers is a useful tip to create realistic testing scenarios.

Unfortunately, this will require some altering of the check answers page. We will need to change the entire page from nunjucks to HTML to make this work.

First, in the code editor navigate to the check answers page. You can now either replace the whole page or you can replace just the table component. Once you have replaced the table, remember to change the content as well as the bits for data playback.

You are able to get the code for this from the GDS design system

For the current example, when a user naviagtes through the changing name journey if they select no on the radio button page, the user will still see the question appear on the check answers page. By adding some simple lines of code we can have this line only appear if the user answers yes on the radio button question.



So at present the prototype currently shows the extra line but this is confusing behaviour.

First of all within the check answers page code find the snippet, which playsback the third question information

Above the that section add the following "{ % if data['choicenewname'] == "Yes" % }" and then underneath the section add in { % endif % }

This will tell the prototype that if the radio button question which has the name "choicenewname" is answered Yes then to go ahead and show the following lines of code. This also means if the user selects No, it will not show it
